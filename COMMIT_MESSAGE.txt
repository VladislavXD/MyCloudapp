fix: Исправление интеграции с Payme и улучшение обработки ошибок

Проблема:
- При оплате через Payme появлялась ошибка "[object Object]"
- Страница Payme не могла обработать платежный запрос

Исправления:
- Добавлен параметр return_url в URL для Payme
- Убрано неправильное кодирование URL (encodeURIComponent внутри base64)
- Добавлена валидация Merchant ID (должен быть 24 символа)
- Добавлено подробное логирование создания платежей
- Улучшены сообщения об ошибках на фронтенде с подсказками
- Добавлена debug информация в API ответы

Новые файлы:
- backend/docs/PAYME_SETUP.md - полная инструкция по настройке
- backend/docs/PAYME_TESTING.md - руководство по тестированию
- backend/test-payme-callback.js - скрипт для тестирования callback API
- PAYME_FIX.md - детальное описание исправлений
- SUMMARY.md - краткое резюме
- QUICK_START.md - инструкция по быстрому старту

Примечание:
Основная причина ошибки - Merchant ID не активирован в системе Payme.
Необходимо получить реальный или тестовый Merchant ID от Payme.

Измененные файлы:
- backend/core/utils/payme-helper.js
- backend/api/payments/payme.js
- app/(user)/checkout.tsx
